# CHANGELOG

## v2.7.0 - 2026-04-08 (Rollback from v3.x)

### 🚨 版本回退

- **回退原因**: 由于 `v3.x` 版本存在严重的性能问题，我们决定将代码库回退到稳定版本 `v2.7.0`。
- **未来规划**: 将从 `v2.7.0` 版本重新规划迭代路线，以确保系统的稳定性和性能。
- **影响**: `v3.x` 版本引入的所有新功能和改进将暂时不可用，直至在新路线图中重新实现。



## v2.7.0 - 2026-03-26 (Original Release)

### 🔧 核心改进

- **修复核心对抗循环**: 实现 `PeripheralAgent` 的 `craft_attack()`、`learn_from_result()`、`discuss_with_peer()`、`learn_from_peer()`、`collaborate_with()` 等关键方法，确保 Agent 间讨论、学习、协作流程能够正常运行。
- **增强中心质检 Agent**: 实现 `CentralAgent.inspect_content()` 方法，完整集成 5 层规则引擎审查流程，返回详细的检测结果（检测层级、命中关键词、置信度等）。
- **完善规则拆解**: 升级 `CentralAgent.refine_rules()` 方法，使用 LLM 对审核规则进行语义拆解，生成变体、同义词和隐喻方式，提升规则引擎的检测能力。
- **版本号全量更新**: 所有代码文件中的版本号从 v2.6.0 同步更新至 v2.7.0。
- **系统健壮性提升**: 改进错误处理机制，确保 LLM 调用失败时系统能够优雅降级，使用默认变体继续运行。

### 📝 文档更新

- 更新 README.md，说明 v2.7.0 的核心改进。
- 补充 CHANGELOG.md 版本记录。

### 🐛 已知问题

- LLM 语义分析层（L5）需要更精细的 prompt 工程以提升中文语境下的理解能力。
- 知识库持久化功能尚未实现，投喂的资料仅存储在内存中。
- Agent 自主搜索能力仍在规划阶段。

## v2.6.0 - 2026-03-22

### 新增功能

- **Agent 攻擊系統 V2 升級**:
  - 攻擊手法分為 3 大類：語言變異類、隱喻影射類、社交黑話類 (移除技術對抗類)。
  - 引入 4 個能力維度：語言變異度、歷史文化厚度、圈層專業度、社會心理操縱度。
  - 新增 72 個 Agent，分為 6 組 (每組 12 人)：G1史鑑組、G2黑話組、G3阴阳组、G4同音组、G5反串组、G6暗流组。
  - 每個 Agent 根據人設分配擅長的攻擊手法和能力。
- **前端資料投喂窗口**:
  - 在網頁左側新增「資料投喂」窗口，用戶可手動投喂攻擊材料、黑話詞典、繞過案例給 Agent 學習。
  - 新增 `/agent/feed` API 接口，用於接收前端投喂的資料並更新 Agent 知識庫。
- **版本號顯示**:
  - 網頁上顯示當前版本號 `v2.6.0`。

### 改进与优化

- **代碼結構調整**:
  - 移除 `attack_knowledge.py`，引入 `attack_knowledge_v2.py` 以支持新的攻擊手法和能力維度定義。
  - `agents.py` 已全面更新，以適應新的 Agent 生成邏輯和攻擊系統。
  - `web_app.py` 已更新，以整合新的 Agent 系統和前端功能。
- **前端界面優化**:
  - 調整了前端佈局，確保「資料投喂」窗口與現有組件協調顯示。

### 其他

- 更新了 `README.md`，反映 `v2.6.0` 的所有變更。
